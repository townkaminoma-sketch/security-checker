## 2. 実装方針

1. **最小差分で追加する**
   - 既存構造を壊さずに変更する

2. **rollback しやすい形で進める**
   - モジュール分離、最小追加、限定ファイル変更を優先する

3. **既存ルールを壊さない**
   - 既存 CLI / config / tests / docs の整合を尊重する
